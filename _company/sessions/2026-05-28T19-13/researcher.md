# 📄 PLV (Potential Loss Value) 데이터 수집 요구사항 명세서 v1.0
**(개발자용 청사진: Mini-Audit 시스템의 입력 데이터 스키마)**

**목표:** JKstory가 고객에게 제시할 '잠재적 재무 손실액(PLV)'을 계산하기 위해 필요한 모든 핵심 데이터를 정의하고, 이 데이터가 갖춰야 할 기술적 최소 요구사항 및 유효성 검사 규칙을 확립한다.
**출처 근거:** CEO 지시 (Mini-Audit 시스템의 입력 필드 정의) & Researcher 개인 메모리 (3PL WMS 시장의 필수 기능 - 청구/수익화).
**톤앤매너:** 권위적, 재무 컨설팅 보고서 형식.

## Ⅰ. 핵심 PLV 계산 구조 및 데이터 흐름 개요

PLV는 단순 비용 합산이 아닌, **[기준 가치] $\times$ [비효율성 지수] + [추가 리스크 요인]**의 복합적인 구조로 산출되어야 합니다. 따라서 입력 필드들은 이 세 가지 축을 커버해야 합니다.

| 분류 | 데이터 그룹 | 포함할 핵심 질문 |
| :--- | :--- | :--- |
| **A. 기준 가치 (Baseline)** | 초기 계약 및 청구 정보 | "만약 문제가 없었다면, 고객은 얼마를 지불했어야 하는가?" |
| **B. 운영 비효율성 (Operational Leakage)** | 물류/공정상의 손실 데이터 | "물류 과정 중 어떤 이유로 자원이나 시간이 낭비되었는가? (수치화 필수)" |
| **C. 재무 리스크 증폭 (Financial Multiplier)** | 외부 요인 및 컴플라이언스 위험 | "이러한 운영 문제는 추가적인 법적/재정적 비용을 야기하는가?" |

## Ⅱ. PLV 데이터 입력 필드 상세 명세서

아래 표는 개발자가 백엔드 API 설계 시 참고해야 할 **필수 입력 파라미터** 목록입니다. 모든 필드는 필수(Required)로 간주하며, 예외 처리가 필요합니다.

| 필드 이름 (API Key) | 설명 및 측정 기준 | 데이터 타입 | 최소/최대 범위 | 유효성 검사 규칙 (Validation Rule) | PLV 계산 기여도 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `base_invoice_amount` | **[A. 기준]** 초기 계약서 또는 최초 청구액의 총 합계. (VAT 포함 최종 금액 기준) | Currency (Decimal) | Min: 1, Max: $10^{9}$ | - 값은 양수여야 함 (`> 0`). <br>- 통화 형식 검증 필수. | **핵심 기반값** |
| `service_period_days` | **[A. 기준]** 청구 기간의 총 일수 (Day). | Integer | Min: 1, Max: 365 | - 반드시 정수여야 함 (`Integer`). <br>- 0일은 불가능. | 비례 계산 기반 |
| `leakage_process_rate` | **[B. 운영]** 공정 단계별 누락 또는 비효율성 비율. (예: 입고 지연, 재고 파손 등) | Float (%) | Min: 0.0%, Max: 100.0% | - 퍼센트(%) 형식 검증 필수. <br>- 100%는 시스템 마비 상태로 간주하여 경고 처리 필요. | **가장 중요한 변수** (비율) |
| `additional_labor_hours` | **[B. 운영]** 비효율성으로 인해 추가 투입된 인력 시간(Man-hour). | Decimal | Min: 0.0, Max: 500.0 | - 소수점 자릿수 허용 (예: 12.5시간). <br>- 음수는 불가능 (`>= 0`). | 직접 비용 증가 |
| `labor_cost_per_hour` | **[B. 운영]** 시간당 평균 인건비 및 간접비(Overhead)를 포함한 단가. | Currency (Decimal) | Min: $5, Max: $1000 | - 값은 양수여야 함 (`> 0`). <br>- 시장 평균 대비 변동성 체크 로직 필요. | 비용 산정 핵심 |
| `compliance_risk_factor` | **[C. 리스크]** 법규 미준수, 서류 누락 등으로 발생하는 잠재적 벌금/조정 비율 계수. (0~1 사이 값) | Float (Ratio) | Min: 0.0, Max: 1.5 | - 범위 제한 필수 (`>= 0` 및 `<= 1.5`). <br>- 이 값이 높을수록 경고색 강조 필요. | 리스크 증폭 계수 |
| `dispute_resolution_cost` | **[C. 리스크]** 분쟁(Dispute) 발생 시 필요한 관리 비용 (변호사, 중재 등). | Currency (Decimal) | Min: $0, Max: $50,000 | - 금액 단위 검증 필수. <br>- 이 값이 0일 경우, 위험성 판단에 신중해야 함. | 고위험 리스크 항목 |

## Ⅲ. PLV 계산 로직 (개발자 가이드라인)

1.  **PLV 기본 공식:**
    $$ \text{PLV} = (\text{base\_invoice\_amount} \times \frac{\text{leakage\_process\_rate}}{100}) + (\text{additional\_labor\_hours} \times \text{labor\_cost\_per\_hour}) + (\text{compliance\_risk\_factor} \times \text{base\_invoice\_amount}) + \text{dispute\_resolution\_cost} $$
2.  **주의 사항:** PLV를 산출할 때는 각 구성 요소(A, B, C)별로 **세부적인 재무적 근거와 원본 증빙 자료의 출처 링크**가 반드시 함께 제시되어야 합니다 (Audit Trail 구현 목표).
