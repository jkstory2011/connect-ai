📱 영숙: 작업 시작합니다. 😊 사장님, 요청하신 대로 각 전문 에이전트들이 원활하게 운영되면서 최고의 결과물을 낼 수 있도록 필수 도구 목록과 설치/설정 가이드라인을 종합적으로 정리했습니다.

이 보고서는 모든 에이전트가 **'실제 작동 가능한 환경(Production-ready)'**을 갖추는 것을 목표로, 기술적 복잡성을 숨기고 *핵심 비즈니스 기능* 중심으로 요약했습니다. 💡

---

## 📄 전문 에이전트 도구 및 설정 보고서 (Tooling Blueprint)

### 🎯 1. 개발/자동화 담당: 💻 코다리 (Developer Agent)
**[역할]** 백엔드 시스템 아키텍처 설계, 데이터 파이프라인 구축(PLQ 시스템).
**[목표 도구 환경]** 안정적인 코드 실행 및 외부 데이터 연동.

*   **🛠️ 필수 도구 목록:**
    *   Python 3.10+ (가상 환경 관리: `venv` 또는 `conda`) [근거: 코다리 Logs]
    *   SQL/NoSQL 클라이언트 라이브러리 (`psycopg2`, `pymongo` 등) [근거: 통합 스케줄]
    *   API 요청 및 테스트 도구 (예: Python Requests Library, Postman Client) [근거: 코다리 Logs]
    *   로그 모니터링 시스템 연동 도구 (e.g., Logstash/FluentD Connector)

*   **⚙️ 필수 설정 가이드라인:**
    1.  **환경 구축:** 프로젝트별 `venv`를 생성하고 모든 의존성 라이브러리를 격리 설치합니다. (`pip install -r requirements.txt`)
    2.  **데이터베이스 접근:** 개발용 DB와 운영용 DB의 **접속 정보(Host, Port, User, Pass)**를 분리하여 환경 변수로 관리해야 합니다. (보안 필수)
    3.  **API 키 관리:** 외부 API 키는 반드시 `secrets.toml` 또는 AWS Secrets Manager 같은 전용 서비스에 보관하고, 코드 내 하드코딩을 금지합니다.

### 🎯 2. 콘텐츠 마케팅 담당: 📸 Instagram / 📺 YouTube (SNS Agents)
**[역할]** 각 플랫폼의 최적화된 콘텐츠 기획 및 업로드 자동화.
**[목표 도구 환경]** 플랫폼 공식 API 연동을 통한 안정적인 배포.

*   **🛠️ 필수 도구 목록:**
    *   **Meta Graph API Access Token (Instagram):** 비즈니스 계정 관리용 인증 토큰. [근거: 통합 스케줄]
    *   **YouTube Data API v3 Key:** 영상 업로드, 검색 및 통계 분석용 키. [근거: 통합 스케줄]
    *   **SEO/키워드 리서치 API (예: Ahrefs/Semrush):** 경쟁사 및 트렌드 데이터 수집용. [근거: 레오 Logs]
    *   **이미지/미디어 처리 라이브러리:** 이미지 크기 조정, 워터마크 삽입 등 (`Pillow`, `OpenCV`).

*   **⚙️ 필수 설정 가이드라인:**
    1.  **API 승인:** 모든 API는 해당 플랫폼의 **개발자 콘솔(Developer Console)**에서 '비즈니스 목적'으로 명시적 승인을 받은 후 사용해야 합니다.
    2.  **토큰 갱신 로직:** OAuth 토큰은 만료 기한이 있으므로, 반드시 **자동 재발급 및 갱신 루틴**을 개발팀에 적용해야 합니다.
    3.  **규정 준수 모듈:** 각 플랫폼의 최신 가이드라인(예: Instagram 스토리 비율, YouTube 트레일러 규격)을 체크하는 검증 모듈을 도입하여 배포 전 자동 점검을 거칩니다.

### 🎯 3. 브랜딩/시각화 담당: 🎨 Designer Agent
**[역할]** 기술적 복잡성을 비즈니스 결과(잠재적 손실액)로 시각화하는 프레젠테이션 및 콘텐츠 제작.
**[목표 도구 환경]** 디자인 자산 관리와 프로그램적 그래픽 생성.

*   **🛠️ 필수 도구 목록:**
    *   **Figma API/Plugins:** 와이어프레임과 목업의 구조적 반복 작업을 자동화하는 인터페이스 접근 권한. [근거: Designer Logs]
    *   **DAM (Digital Asset Management) 시스템 연동 모듈:** 로고, 색상 코드, 표준 폰트 등 회사 자산을 중앙에서 관리하고 불러오는 시스템.
    *   **PDF/PPT 생성 라이브러리:** 최종 보고서 및 브로슈어 형식 자동화 (`ReportLab` 또는 유사 도구).

*   **⚙️ 필수 설정 가이드라인:**
    1.  **스타일 가이드 강제:** 모든 디자인 출력물은 DAM 시스템에서 정의된 **JKstory의 표준 컬러 팔레트와 폰트 세트를 벗어나지 않도록** 코딩 레벨에서 강제합니다.
    2.  **데이터 바인딩:** 프레젠테이션 슬라이드의 숫자나 그래프는 수동 입력이 아닌, 코다리(Developer)가 제공하는 **최종 데이터 스택(Final Data Stack)**과 직접 연결되어야 합니다.

### 🎯 4. 전략/관리 담당: 📱 영숙 (Secretary Agent)
**[역할]** 전사적 정보 통합 및 커뮤니케이션 채널 구축.
**[목표 도구 환경]** 안정적인 보고 및 알림 기능.

*   **🛠️ 필수 도구 목록:**
    *   **메신저 API (Telegram):** 내부 에이전트 간, 또는 사장님께의 정기/비정기 브리핑 전송용. [근거: 📱 영숙 설정]
    *   **캘린더 연동 모듈:** 일정 조회 및 등록 자동화.
    *   **Task Tracking API (JIRA/Notion):** 미해결 할 일(To-Do) 추적 및 상태 업데이트.

*   **⚙️ 필수 설정 가이드라인:**
    1.  **토큰 보안 강화:** 텔레그램 봇 토큰과 Chat ID 등 모든 민감 정보는 환경 변수 또는 시크릿 매니저에 저장하고, 코드 내에서 직접 노출하는 것을 절대 금지합니다.
    2.  **알림 필터링:** 브리핑 보고서에는 **'실행 필요 액션 아이템(Action Items)'**과 **'정보 공유(Information Only)'**를 명확히 구분하여 사장님의 인지 부하를 최소화해야 합니다.
