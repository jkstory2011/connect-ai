### 💻 PLV 프로세스 트래킹 대시보드: 인터랙션 디자인 가이드 (Wireframe Concept)

**[전체 구조]**: 좌측에는 데이터 흐름도(Flow Diagram, 배경 패턴 유지), 중앙에 단계별 진단 모듈(Detail View), 우측에 요약된 위험 리스크 및 조치 권고 사항을 배치합니다.

#### 1. 초기 로딩 및 전체 뷰 (The Overview - 공포 유발)
*   **목표:** 사용자가 처음부터 시스템의 복잡성과 잠재적 손실의 심각성을 느끼게 합니다. [근거: Self-RAG, 전문성 강조]
*   **레이아웃:** 대시보드의 가장 상단은 **'Audit Trail Schema Flow Chart'**가 흐릿한 배경 패턴으로 깔립니다. (복잡하게 얽힌 데이터 파이프라인 다이어그램).
*   **핵심 요소:** 주요 처리 단계(Ingestion $\to$ Validation $\to$ Calculation)를 순차적으로 보여주는 큰 화살표와 박스들이 배치됩니다.
*   **[Risk Amber] 인터랙션 (Phase 1):** 시스템이 데이터를 로딩하는 과정에서, **Validation 단계의 특정 노드**에 데이터 불일치 문제가 발견되면, 해당 노드의 테두리가 `#C94A1B` (Risk Amber)로 깜빡이며 시각적 경고가 발생합니다.
    *   **UI 반응:** 전체 대시보드가 미묘하게 어두워지며(Ambient Drone 사운드 연상), 중앙에 "⚠️ Critical Data Discrepancy Detected"라는 경고 배너가 강렬하게 등장합니다. [근거: Potential Loss Shielding]

#### 2. 위험 진단 및 상세 조사 (The Deep Dive - 분석 과정 강조)
*   **목표:** 사용자가 클릭한 '위험 지점'에 대해, **무엇이 잘못되었는지(What)**와 **어떤 근거로 판단했는지(Why)**를 명확히 제시합니다.
*   **트리거:** 사용자가 [Risk Amber]가 발생한 노드 (예: Validation 단계의 `Data Inconsistency` 지점)를 클릭하면, 중앙 디테일 뷰가 확장됩니다.
*   **레이아웃 변화:** 중앙 디테일 뷰는 **'Problem Statement'** 영역과 **'Audit Evidence'** 영역으로 분리됩니다.
    *   **Problem Statement (위험 설명):** "3개의 마이크로서비스 간의 연결 고리가 끊어져 있습니다."와 같은 현빈 에이전트가 정의한 Pain Point를 텍스트로 제시합니다.
    *   **Audit Evidence (증거 자료):** 이 영역에 실제 데이터 샘플(A사 기록, B사 장부)을 나란히 배치하고, **불일치하는 필드만 하이라이트** 합니다.

##### ✨ Source Grounding URL 인터랙션 설계
1.  **위치:** `Audit Evidence` 영역의 불일치한 값 옆에 작은 아이콘 (🌐)과 함께 'Source Discrepancy'라는 텍스트가 붙습니다.
2.  **행동:** 사용자가 이 아이콘이나 주변의 데이터 필드를 **호버(Hover)**하거나 클릭하면, 해당 데이터 포인트의 출처 정보와 원본 데이터를 보여주는 작은 모달 창이 팝업 됩니다.
3.  **모달 내용:** `Source Name: [A사 시스템]`, `Timestamp: YYYY-MM-DD HH:MM:SS`, **`Source Grounding URL: (링크)`**. 이 URL은 클릭 가능한 상태로 제공됩니다. [근거: Potential Loss Shielding]

#### 3. 해결책 제시 및 전환 (The Solution - 희망 부여)
*   **목표:** 공포(Risk Amber)에서 JKstory의 권위와 솔루션으로 자연스럽게 전환시킵니다. [근거: Self-RAG, Phase 2]
*   **레이아웃 변화:** 'Audit Evidence' 아래에 강한 구분선(`Security Blue`, `#007BFF`)이 가로지르며 **'JKstory Shielding Solution'** 섹션으로 전환됩니다.
*   **핵심 기능 배치:** 이 영역에는 JKstory의 핵심 기능을 3개의 아이콘 형태로 배치합니다.
    1.  **AI 리스크 예측 엔진:** (아이콘) 잠재적 손실액을 사전에 포착하는 과정 시뮬레이션을 작은 차트(Before/After)로 보여줍니다.
    2.  **Source Grounding 시스템:** (아이콘) "데이터 파편화 방지"라는 서브 텍스트와 함께, 모든 데이터에 출처 태그가 붙는 애니메이션을 짧게 재생합니다.
    3.  **프로세스 자동화 워크플로우:** (아이콘) 사람이 개입할 필요 없이 시스템이 자동으로 연결 고리를 복구하는 플로우차트를 보여줍니다.
