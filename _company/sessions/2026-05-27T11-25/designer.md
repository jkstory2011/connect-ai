## 📊 JKstory 데이터 시각화 컴포넌트 3종 (Potential Loss Focus)

### 💡 Concept 1: The "Potential Loss Gap" (재무 손실 격차 비교 다이어그램)
가장 임팩트가 강한, 'Before & After' 구조의 비교 자료입니다. 고객에게 '얼마나 많이 잃을 수 있었는지'를 시각적으로 충격 요법을 가하는 것이 목표입니다.

*   **목표 메시지:** JKstory 도입 전후의 재무적 격차(Gap)가 얼마나 큰지, 숫자로 증명한다.
*   **레이아웃:** 좌우 대칭 구조 (Before | After).
*   **핵심 요소:** 거대한 화살표와 대비되는 색상 영역.

| 디자인 항목 | 상세 지침 | 스타일 코드 | 비고/효과 |
| :--- | :--- | :--- | :--- |
| **헤드라인** | Potential Loss Gap: 데이터 불일치가 만든 기회비용 | H2급, 굵게(Bold) 처리. 신뢰감을 주는 세리프체 사용 권장 (예: Noto Serif KR). | 금융 보고서의 무게감 부여. |
| **Before (위험 상태)** | '미관리 리스크' 구간을 표현하는 막대 차트/영역. | 배경색: `#FFB74D` (Risk Amber, 경고) / 텍스트: `#8D0000` (Deep Red). | 손실액에 비례하여 영역의 폭이 커질수록 위험도가 높음을 시각화. |
| **Gap 표시** | Before와 After 사이를 가르는 거대한 '재무 보호막' 화살표. | 색상: `#1A3E6D` (JK Deep Blue). 투명도를 70% 적용하여 깊이감을 부여. | 이 부분이 JKstory의 핵심 기능(Shielding)을 상징함. |
| **After (솔루션 상태)** | '관리되는 리스크' 구간을 표현하는 작은 막대 차트/영역. | 색상: `#4CAF50` (Success Green). 안정감을 주는 색상. | Potential Loss가 90% 이상 감소했음을 시각적으로 압도한다. |
| **데이터 레이블** | Before: $XX Million (잠재적 손실) / After: $Y Thousand (잔여 리스크) | 숫자를 가장 크게, 강조된 글꼴로 배치. 단위를 명확히 표기. | 'Million'과 'Thousand'의 차이를 시각적으로 대비시켜야 함. |

### 💡 Concept 2: The "Discrepancy Flow Map" (시스템 흐름 오류 지도)
단순히 목록을 나열하는 대신, 데이터가 어느 과정에서 어떻게 깨졌는지를 보여주는 프로세스 흐름도입니다. 이는 JKstory의 *분석 깊이*를 증명합니다.

*   **목표 메시지:** 문제는 '데이터 자체'가 아니라 '시스템 간의 연결 구조(Process Flow)'에 있다.
*   **레이아웃:** 왼쪽에서 오른쪽으로 진행하는 3~4단계 플로우 차트 형태.
*   **핵심 요소:** 정상 경로(Green)와 오류 발생 지점(Red Alert).

| 디자인 항목 | 상세 지침 | 스타일 코드 | 비고/효과 |
| :--- | :--- | :--- | :--- |
| **플로우 단계 (System)** | 1. Source Data Input $\to$ 2. OMS Processing $\to$ 3. ERP Reconciliation $\to$ 4. Final Report Output | 박스 형태의 단계별 구분. 각 시스템 이름에 아이콘(Icon)을 부여하여 전문성 강화. | 프로세스가 한눈에 들어오게 구조화. |
| **정상 데이터 흐름** | 화살표가 부드럽고 일관되게 연결됨. | 색상: `#607D8B` (Gray-Blue). 신뢰성과 연속성을 표현. | 자연스러운 데이터 이동을 시각화. |
| **불일치 지점 (Discrepancy)** | 특정 단계(예: 2 $\to$ 3 사이)의 화살표가 '꺾이거나 끊어지며' 경고 영역으로 진입함. | 색상: `#E53935` (Alert Red). 강력한 시각적 중단 효과. | 지점 위에 `[!] Discrepancy Detected!` 태그와 함께 해당 오류 유형을 텍스트로 명시. |
| **리스크 상세 카드** | 각 불일치 지점 아래에 작은 '카드' 형태로 리스크 요약 제시. | 구조: 필드명 / 원본값 $\to$ 예상규칙 / 위반사유 (작은 글씨로 배치). | 데이터의 복잡성을 학술적인 느낌으로 정리하여 권위를 높임. |

### 💡 Concept 3: The "Risk Heatmap Scorecard" (위험도 지수 종합 평가)
가장 빠르고 직관적으로 '당신의 회사는 위험하다'는 메시지를 던지며, 전반적인 컨설팅의 총체적 결론을 제시할 때 사용합니다.

*   **목표 메시지:** 복합적인 리스크를 단 하나의 수치(Index Score)로 요약하여 심리적 압박감과 해결 욕구를 자극한다.
*   **레이아웃:** 대시보드 형태의 게이지 미터와 4분면 매트릭스 조합.
*   **핵심 요소:** 시각적인 '위험 점수'의 측정.

| 디자인 항목 | 상세 지침 | 스타일 코드 | 비고/효과 |
| :--- | :--- | :--- | :--- |
| **메인 게이지 미터** | 현재 고객사의 리스크 레벨을 표시하는 반원형 게이지(Gauge Meter). | 기준점: 0% (안정) $\to$ 100% (위험). | 포인터를 `#FFB74D` (Risk Amber) 영역에 위치시켜 불안감을 최고조로 만든다. |
| **스코어링 시스템** | 리스크 점수(Potential Loss 기반 가중치 합산)를 명확한 숫자로 제시. | 예: "Overall Risk Index: 82/100" | 수치를 통해 객관적이고 과학적인 분석임을 강조. |
| **4분면 매트릭스** | X축 (발견된 불일치의 개수), Y축 (불일치로 인한 잠재 손실 규모)를 설정하여 위치시킴. | 영역 색상: 좌측 하단(안전, Green) $\to$ 우측 상단(매우 위험, Red). | 고객의 현재 위치가 매트릭스의 어느 지점에 놓여있는지 시각적으로 보여주어 위기감을 극대화한다. |
| **해결책 가이드** | 이 분석을 해결할 수 있는 JKstory의 핵심 기능 3가지 아이콘 배치. | 각 아이콘 아래에 간략한 설명과 함께 `[근거: Potential Loss Shielding]` 태그를 작게 추가. | 공포(Fear) $\to$ 희망(Hope)으로 자연스럽게 전환시키는 역할. |
