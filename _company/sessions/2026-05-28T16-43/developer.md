# 🛡️ JKstory Mini-Audit 시스템: 기술적 신뢰성 검증 (Technical Validation)

**[타겟 독자]:** 이 자료는 '운영 효율'에만 초점을 맞춘 기존 솔루션의 한계를 이해하는, 재무 건전성을 중요하게 생각하는 사업주를 대상으로 합니다.
**[핵심 메시지]:** JKstory는 복잡하고 시간이 오래 걸리는 리스크 계산을 **분할 처리(Async)**하여, 단 하나의 데이터 오류도 놓치지 않고 **출처가 명시된(Source Grounding)** 완벽한 보고서를 약속합니다.

### 1. 작동 원리 이해하기: 비동기 아키텍처란 무엇인가요? (The Concept)

**[비유]:** 기존 시스템이 '은행 창구 직원'이라면, JKstory는 '전담 전문 감사팀(Audit Team)'과 같습니다.
*   **문제점:** 만약 고객님께서 "지난 3년간 모든 거래 내역을 분석해주세요"라고 요청하면, 일반적인 웹사이트(기존 시스템)는 그 복잡한 계산을 하다가 **타임아웃(Time-out)**되거나, 중간에 서버가 과부하로 멈춰버릴 위험이 있습니다.
*   **JKstory의 해결책 (비동기 워커):** 저희는 요청을 받으면 먼저 전문 감사팀에게 '작업 지시서'를 던집니다. 그리고 고객님께는 "지금 분석 작업을 시작했습니다. 약 O분 뒤에 결과를 받아보세요."라고 안내합니다. 이 과정은 시스템 자체의 부하가 아닌, **전문적인 백그라운드 전용 워커(Worker)**에서 진행됩니다.
*   **결과:** 서버는 항상 빠르고 안정적이며, 가장 복잡한 계산도 시간제한 없이 끝까지 처리할 수 있습니다.

### 2. Mini-Audit의 데이터 흐름 (The Data Flow: Request $\to$ Result)

Mini-Audit 과정은 다음의 3단계 순차적 과정을 거치며 진행됩니다. 이 구조 자체가 JKstory가 **'끊기지 않는 신뢰성'**을 제공한다는 증거입니다.

| 단계 | 주체 (역할) | 프로세스 설명 | 사용자 체감 경험 (UX) |
| :---: | :---: | :--- | :--- |
| **1️⃣ 요청 및 큐잉 (Request)** | **API Gateway** (최전방 게이트) | 사용자가 분석 데이터를 업로드하고 '분석 시작' 버튼을 누릅니다. API는 이 요청을 받아 즉시 백그라운드 **Task Queue(작업 대기열)**에 작업을 등록합니다. (이때 응답 지연 없음!) | "요청 접수 완료. 현재 작업 중입니다." 메시지 확인. |
| **2️⃣ 분석 및 검증 (Processing & Grounding)** | **Worker Task** (전담 워커) | 전담 워커가 대기열에서 작업을 가져와 실행합니다. 이 단계에서 **(1) 데이터 통합 $\to$ (2) PLV 계산 $\to$ (3) Source Grounding 체크**를 수행하며, 모든 수치에 대한 출처(Source of Truth)를 찾아내고 리스크 레벨을 판단합니다. | 시스템이 아무것도 하지 않아도, 내부적으로 가장 복잡한 검증 작업을 수행 중입니다. |
| **3️⃣ 결과 전송 및 보고 (Delivery)** | **Result API** (결과 조회 API) | 계산이 완료되면 워커는 최종 보고서를 저장하고, 사용자에게 '완료' 알림을 보냅니다. 사용자는 별도 요청으로 결과를 받아보고, 즉시 분석 리포트를 열람합니다. | "분석 완료! [PLV] 보고서 확인하기" 버튼 활성화 및 데이터 시각화. |

### 3. 핵심 API 엔드포인트 상세 (Technical Blueprint)

이러한 흐름을 뒷받침하는 백엔드 기술 스택은 다음과 같습니다. 이 구조는 단지 기능 구현이 아니라, **안정성과 확장성**을 보장합니다.

| 엔드포인트 | 역할 (Function) | 비즈니스적 가치 (Why it Matters) |
| :--- | :--- | :--- |
| `POST /api/v1/audit/request` | **[요청 시작]** 분석 작업을 백그라운드 워커에 등록합니다. | **🚀 즉각성 확보:** 요청을 받자마자 큐(Queue)에 넣고 응답하므로, 사용자는 시스템이 멈춘다고 느끼지 않습니다. (안정적인 사용자 경험 제공) |
| `GET /api/v1/audit/{job_id}/status` | **[진행 상황 확인]** 특정 분석 작업의 현재 상태(대기 중, 진행 중, 완료)를 조회합니다. | **🔍 투명성 확보:** 고객은 언제쯤 결과가 나올지 정확히 알 수 있습니다. '분석이 멈췄나?' 하는 불안감을 해소합니다. |
| `GET /api/v1/audit/{job_id}/result` | **[최종 보고서 전송]** 작업이 완료된 후, 최종적으로 PLV 및 리스크 분석 결과를 받아옵니다. | **✅ 신뢰성 완성:** 모든 데이터가 통합되고 검증된 '진짜 결과'만 전달받습니다. 사후 예측 가능성을 증명합니다. |
