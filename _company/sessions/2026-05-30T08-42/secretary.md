# 🚀 JKstory 데이터 레이어 구축 프로젝트 차터 (v1.0)

## 🎯 프로젝트 목표
'물류 운영 효율화' 컨설팅을 넘어, 고객의 '잠재적 재무 리스크(Potential Loss)'를 측정하고 이를 근거 기반으로 제시하는 **통합 시스템 아키텍처**를 구축한다. 이 PoC는 고가치 플랜($299+/월) 판매에 필요한 기술적 신뢰성을 확보하는 것이 핵심이다.

## 🗺️ 시스템 구조 (Reference: 코다리)
1.  **Ingress:** `data_collector` (외부 API 연동 및 웹훅 수신)
2.  **Processing:** `worker_queue` (데이터 변환, 검증, KPI 계산 로직 실행)
3.  **Storage:** PostgreSQL (영속적 데이터 저장소)
4.  **Output/API Gateway:** FastAPI를 통한 결과 조회 API 제공

## 📅 마일스톤 및 액션 플랜 (Timeline & Milestones)

### [Phase 1: Data Foundation] - 목표 완료일: D+3
*   ✅ **액션:** 현빈 님이 정의한 KPI(LTV, MRR 등) 기반의 최종 데이터 스키마 확정.
*   👤 **책임자:** 코다리 (개발), 현빈 (기획 검증).
*   🛠️ **산출물:** PostgreSQL 초기 환경 설정 및 `users`, `subscriptions`, `transactions` 테이블 구조 완성.

### [Phase 2: Core Pipeline Build] - 목표 완료일: D+5
*   ✅ **액션:** 데이터 수집(Ingress) $\rightarrow$ 처리(Processing) $\rightarrow$ 저장(Storage)의 자동화된 흐름 테스트.
*   👤 **책임자:** 코다리 (개발 전담).
*   🛠️ **산출물:** 핵심 파이프라인 작동 시뮬레이션 보고서 및 에러 처리 로직 구현.

### [Phase 3: PoC Validation & Reporting] - 목표 완료일: D+7
*   ✅ **액션:** 구축된 데이터를 활용하여 'Potential Loss'를 계산하고, 이를 비즈니스 관점에서 가장 설득력 있게 시각화하는 데모 완성.
*   👤 **책임자:** 현빈 (비즈니스 검증), Designer (시각화 기획).
*   🛠️ **산출물:** 고객 대상 프레젠테이션용 MVP 대시보드 작동 버전 및 최종 ROI 계산 로직 정의.
